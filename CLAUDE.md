# HackerWeb

A simply readable Hacker News web app (https://hackerwebapp.com/). Static site, no backend — fetches from the unofficial node-hnapi. Deployed via GitHub Pages (`gh-pages` branch).

## Stack

- Vanilla JavaScript (ES modules, no framework). Two themes: `hw-ios` (iOS Mobile Safari) and `hw-web` (everything else); `hw-ios-2` is a newer iOS variant.
- Templating: Hogan.js (mustache) — pre-compiled templates in `assets/js/templates.js`.
- Storage: Amplify.Store. Routing: ruto.js (`location.hash`).
- PWA: `service-worker.js`, `manifest.json` / `manifest.webapp`, `chrome-web-app/`.
- Build: Vite. Formatting: Prettier.

## Layout

- `index.html`, `404.html`, `options.html` — entry pages.
- `assets/` — **source** CSS, JS, images, templates (edit here).
- `js/` — **built/minified** bundles (`hw-ios.min.js`, `hw-web.min.js`, etc.) + `hnapi-worker.js`. Generated by `npm run build`; do not hand-edit.
- `vite.config.js` — Vite build config.
- `.prettierrc` — Prettier config.
- `sources/` — Affinity Designer / xar source art.
- `icons/`, `screenshots/`, `promo-images/`, `startups/` — assets for stores/PWA.
- `script/` — misc shell/utility scripts.

## Common commands

- `npm install` — install deps.
- `npm run dev` — Vite dev server (serves source files directly with HMR).
- `npm run build` — production build (bundles `assets/js/*` → `js/*`).
- `npm run preview` — preview the production build locally.
- `npm run format` — format source files with Prettier.
- `npm run format:check` — check formatting without modifying files.
- `npm run deploy` — merge `master` into `gh-pages` and push.

## Conventions / gotchas

- Source JS uses ES modules (`import`/`export`). Each theme entry file (`hw-web.js`, `hw-ios.js`, `hw-ios-2.js`) imports shared code from `hw.js` and theme-specific libraries.
- Edit sources under `assets/`, then run `npm run build` to regenerate `js/*` bundles before committing.
- iOS vs web themes are separate bundles — changes to shared behavior may need to land in both (`hw-web.js`, `hw-ios.js`, `hw-ios-2.js` for event handlers; `hw-web.css`, `hw-ios.css`, `hw-ios-2.css` for styles). Shared logic lives in `hw.js`.
- Event binding differs per theme: `hw-web.js` uses `ibento()`, `hw-ios.js` and `hw-ios-2.js` use `tappable()`.
- `index.html` dynamically loads the right theme bundle based on user agent. In dev mode it loads source files; in production it loads built bundles from `js/`.
- Analytics is Cloudflare Analytics (Sentry/GA were removed — see recent commits).

## Comments collapse/expand

- **Children toggle (`comments-toggle` button):** Every nested `<ul>` with ≥3 descendant comments gets a `[N replies]` button. Subtrees start collapsed. Clicking toggles `display:none` on the `<ul>` and flips a `.collapsed` class on the button. Subtrees with <3 descendants render inline with no toggle.
- **Parent collapse (`collapse-parent` button):** A double-chevron-up icon replaces the old decorative `comment-arrow` on nested `<li>`s (hidden on top-level comments via CSS). Clicking walks up to the grandparent comment, hides the parent `<ul>`, marks the grandparent's `comments-toggle` as `.collapsed`, and scrolls the toggle into view.
- Key files: `assets/templates/comments.mustache`, `assets/js/hw.js` (`toggle`, `collapseParent`), theme CSS files (`.collapse-parent` styles), theme JS files (click/tappable handlers).
