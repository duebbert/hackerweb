# HackerWeb

A simply readable Hacker News web app (https://hackerwebapp.com/). Static site, no backend. Deployed via Cloudflare Pages (auto-builds from `master` on push).

## Stack

- Vanilla JavaScript (ES modules, no framework). Single responsive theme with CSS animations and wide-screen dual-pane layout.
- Templating: JSX-like functions in `src/js/templates.jsx` using string concatenation with `escapeHTML()` for XSS safety.
- Storage: `store.js` — lightweight wrapper around `localStorage`/`sessionStorage` with TTL support.
- Routing: `router.js` — minimal hash-based router using `hashchange` events.
- API: `hnapi.js` — single Algolia HN Search API source for both stories and comments. Uses `fetch()` + `AbortController`.
- PWA: `service-worker.js`, `manifest.json` / `manifest.webapp`.
- Build: Vite (standard SPA mode). Formatting/linting: Biome.

## Layout

- `index.html` — main entry page (at project root, processed by Vite).
- `src/js/main.js` — single entry point (events, animations, scroll persistence). Imports CSS.
- `src/js/hw.js` — core shared logic (news, comments, routing, PubSub, offline preloader).
- `src/js/templates.jsx` — template functions for rendering HTML.
- `src/js/libs/` — `store.js`, `router.js`, `hnapi.js`.
- `src/css/hw.css` — single unified CSS with custom properties for dark mode.
- `src/images/` — image assets referenced by CSS.
- `public/` — static files copied as-is to `dist/` (icons, manifests, service worker, 404.html, options.html).
- `dist/` — production build output. Generated by `npm run build`; do not hand-edit. Not checked into git.

## Commands

- `npm run dev` — Vite dev server with HMR.
- `npm run build` — production build (bundles JS, CSS, images → `dist/`).
- `npm run preview` — preview the production build locally.
- `npm run format` — format with Biome.
- `npm run format:check` — check formatting without modifying.
- `npm run lint` / `npm run lint:fix` — lint with Biome.
- `npm run check` / `npm run check:fix` — format + lint combined.

## Versioning

- Date-based version in `package.json` `"version"` field, format `YYYY.MM.DD` (e.g. `2026.04.11`).
- If multiple commits happen on the same day, append a sub-counter: `2026.04.11.2`, `2026.04.11.3`, etc.
- Bump the version in `package.json` on every commit.
- Vite injects the version at build time via `__APP_VERSION__`; it's displayed on the About page.

## Conventions

- Edit sources under `src/`, then `npm run build` to regenerate `dist/`.
- CSS uses custom properties (`:root` variables) for theming. Dark mode via `@media (prefers-color-scheme: dark)`.
- Event binding uses native event delegation: `document.body.addEventListener()` + `event.target.closest()`.
- View transitions use CSS `@keyframes` animations. Wide screen (>736px) uses a dual-pane layout.
- Stories and comments both come from Algolia (`hn.algolia.com/api/v1/`). Stories use `search?tags=front_page`, comments use `items/{id}`.
- Comment cache (`hacker-item-{id}`, 24h TTL) is invalidated when the story list shows a different `comments_count` than the cached value.
- The offline preloader eagerly fetches and caches comments for all visible stories in the background.

## Comments collapse/expand

- **Children toggle (`comments-toggle`):** Nested `<ul>` with ≥3 descendant comments gets a `[N replies]` button. Subtrees start collapsed.
- **Parent collapse (`collapse-parent`):** Double-chevron-up icon on nested `<li>`s. Walks up to grandparent comment, collapses the parent subtree, scrolls into view.
- Key files: `templates.jsx` (comment HTML), `hw.js` (`toggle`, `collapseParent`), `hw.css` (styles), `main.js` (click handlers).
