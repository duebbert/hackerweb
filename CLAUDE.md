# HackerWeb

A simply readable Hacker News web app (https://hackerwebapp.com/). Static site, no backend — fetches from the unofficial node-hnapi. Deployed via GitHub Pages (`gh-pages` branch).

## Stack

- Vanilla JavaScript (ES modules, no framework). Single responsive theme with CSS animations and wide-screen dual-pane layout.
- Templating: Plain JS functions using string concatenation with `escapeHTML()` for XSS safety (`assets/js/templates.js`).
- Storage: Lightweight `store.js` wrapper around `localStorage`/`sessionStorage` with TTL support.
- Routing: `router.js` — minimal hash-based router using `hashchange` events.
- API: `hnapi.js` — uses `fetch()` + `AbortController` with URL failover cascade.
- PWA: `service-worker.js`, `manifest.json` / `manifest.webapp`, `chrome-web-app/`.
- Build: Vite. Formatting: Prettier.

## Layout

- `index.html`, `404.html`, `options.html` — entry pages.
- `assets/` — **source** CSS, JS, images (edit here).
- `assets/js/hw.js` — core shared logic (news, comments, routing, PubSub).
- `assets/js/hw-unified.js` — single entry point (events, animations, scroll persistence).
- `assets/js/templates.js` — template functions for rendering HTML.
- `assets/js/libs/` — `store.js`, `router.js`, `hnapi.js`.
- `assets/css/hw.css` — single unified CSS with CSS custom properties for dark mode.
- `js/` — **built/minified** bundle (`hw-unified.min.js`). Generated by `npm run build`; do not hand-edit.
- `vite.config.js` — Vite build config (single entry point).
- `.prettierrc` — Prettier config.
- `sources/` — Affinity Designer / xar source art.
- `icons/`, `screenshots/`, `promo-images/`, `startups/` — assets for stores/PWA.
- `script/` — misc shell/utility scripts.

## Common commands

- `npm install` — install deps.
- `npm run dev` — Vite dev server (serves source files directly with HMR).
- `npm run build` — production build (bundles `assets/js/*` → `js/hw-unified.min.js`).
- `npm run preview` — preview the production build locally.
- `npm run format` — format source files with Prettier.
- `npm run format:check` — check formatting without modifying files.
- `npm run deploy` — merge `master` into `gh-pages` and push.

## Conventions / gotchas

- Source JS uses ES modules (`import`/`export`). `hw-unified.js` imports from `hw.js`, which imports libs and templates.
- Edit sources under `assets/`, then run `npm run build` to regenerate `js/*` before committing.
- CSS uses custom properties (`:root` variables) for light/dark theming. Dark mode is handled via `@media (prefers-color-scheme: dark)` overriding `:root` values.
- Event binding uses native event delegation: `document.body.addEventListener()` + `event.target.closest()`.
- View transitions use CSS `@keyframes` animations (slide left/right, slide up/down). Wide screen (>736px) uses a dual-pane layout with overlay for About dialog.
- `index.html` loads `assets/css/hw.css` statically and dynamically loads the JS entry point (source in dev, built in production).
- Analytics is Cloudflare Analytics (Sentry/GA were removed — see recent commits).

## Comments collapse/expand

- **Children toggle (`comments-toggle` button):** Every nested `<ul>` with ≥3 descendant comments gets a `[N replies]` button. Subtrees start collapsed. Clicking toggles `display:none` on the `<ul>` and flips a `.collapsed` class on the button. Subtrees with <3 descendants render inline with no toggle.
- **Parent collapse (`collapse-parent` button):** A double-chevron-up icon on nested `<li>`s (hidden on top-level comments via CSS). Clicking walks up to the grandparent comment, hides the parent `<ul>`, marks the grandparent's `comments-toggle` as `.collapsed`, and scrolls the toggle into view.
- Key files: `assets/js/templates.js` (comment HTML generation), `assets/js/hw.js` (`toggle`, `collapseParent`), `assets/css/hw.css` (`.collapse-parent` styles), `assets/js/hw-unified.js` (click handlers).
